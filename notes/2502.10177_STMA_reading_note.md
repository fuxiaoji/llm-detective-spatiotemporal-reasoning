# STMA 论文阅读笔记

论文：**STMA: A Spatio-Temporal Memory Agent for Long-Horizon Embodied Task Planning**

本地文件：

- `papers/2502.10177_STMA_Spatio_Temporal_Memory_Agent.pdf`
- `papers/2502.10177_STMA_Spatio_Temporal_Memory_Agent.txt`

网页：

- `https://arxiv.org/html/2502.10177v2`

## 一句话总结

STMA 提出一个带有**时空记忆**的 LLM agent 框架，用 temporal memory 记录历史轨迹，用 dynamic knowledge graph 表示空间关系，再通过 planner-critic 闭环来生成、检查和修正行动计划。

这篇论文对我们很有用，因为它证明了：

> 对长程动态任务来说，光让 LLM 用普通历史记录或 CoT 推理不够；显式的时空记忆和验证模块能明显提升任务完成率。

## 研究问题

论文关注 embodied intelligence 中的长程任务规划问题。Agent 在动态环境里执行任务时，需要：

- 记住之前发生过什么；
- 理解房间、物体和路径的空间关系；
- 根据环境变化调整计划；
- 避免执行已经过时或不合法的行动。

作者认为现有 LLM agent 有两个核心不足：

1. **Memory limitations**：LLM 很难长期追踪任务上下文和历史信息。
2. **Spatio-temporal reasoning deficits**：LLM 缺少显式建模时空关系的能力。

## 方法：STMA 框架

STMA 包含三大组件：

### 1. Temporal Memory

Temporal memory 保存历史 action-observation 记录，并通过 summarizer 压缩成 temporal belief。

它解决的问题是：

- 原始历史太长，会让 prompt 膨胀；
- 历史里有很多无关信息；
- LLM 很难直接从长历史中提取当前任务需要的信息。

对我们课题的启发：

侦探故事里可以把时间线、人物行动、证词出现顺序整理成 **temporal belief**，而不是把全文反复塞给模型。

### 2. Spatial Memory with Dynamic KG

Spatial memory 用动态知识图谱保存空间关系。

基本流程：

1. 从 temporal belief 中抽取空间关系 triples。
2. 更新 dynamic KG。
3. 根据当前任务检索相关子图。
4. 用 relation aggregator 把 triples 转成自然语言形式的 spatial belief。

论文特别提到，直接把 triples 列表塞给 LLM 可能效果不好，所以 relation aggregator 会把关系整理成更容易理解的自然语言，并进行简单空间推理。例如：

如果 A 在 B 西边，B 在 C 西边，则可以推出 A 在 C 西边。

对我们课题的启发：

我们可以把 spatial memory 扩展成 **spatio-temporal evidence graph**：

- Person：嫌疑人、受害者、证人
- Time：某时刻、事件先后顺序
- Location：房间、街道、案发现场
- Object：凶器、信件、手机、钥匙
- Event：进入、离开、看到、移动、死亡
- Testimony：某人说了什么
- Conflict：证词矛盾、不在场证明冲突

### 3. Planner-Critic

Planner 根据 temporal belief、spatial belief 和当前观察生成 subgoal 和 action sequence。

Critic 在每一步执行前检查行动是否：

- 与 temporal belief 一致；
- 与 spatial belief 一致；
- 符合当前环境；
- 没有过时或不合法。

如果 critic 判断行动不合理，就让 planner 重新生成后续计划。

对我们课题的启发：

侦探推理里可以把 planner-critic 改成：

- **Reasoner / Planner**：提出可能凶手和解释路径。
- **Critic / Verifier**：检查解释是否和证据图一致，是否违反时间线、地点、物品状态或证词。

这样就不是单纯“让模型多想”，而是让模型在证据图约束下修正推理。

## 实验设置

论文在 TextWorld 环境中测试 cooking tasks。

任务特点：

- Agent 随机出现在多个房间中；
- 需要探索环境，找到厨房；
- 根据菜谱寻找食材和厨具；
- 食材和工具可能不在厨房；
- 任务有不同难度等级；
- 每个任务限制 50 turn。

难度设置：

| Level | Rooms | Ingredients to Find | Recipe Steps | Total Score |
|---|---:|---:|---:|---:|
| 1 | 6 | 0 | 4 | 4 |
| 2 | 9 | 1 | 6 | 7 |
| 3 | 9 | 2 | 8 | 10 |
| 4 | 12 | 3 | 10 | 13 |

对比 baseline：

- ReAct
- Reflexion
- AdaPlanner
- STMA

使用模型：

- GPT-4o
- Qwen2.5-72b-instruct

## 主要结果

论文报告 STMA 在不同难度上总体优于 baseline。

关键结果：

- 对 GPT-4o，STMA 相比最佳 baseline：
  - Success Rate 平均提升 12.5%
  - Average Score 平均提升 11.15%

- 对 Qwen2.5-72b-instruct，STMA 相比最佳 baseline：
  - Success Rate 平均提升 31.25%
  - Average Score 平均提升 24.7%

这说明结构化时空记忆对开源模型帮助更明显。

## 消融实验重点

消融实验对我们很重要，因为它说明每个模块是否真的有用。

### 去掉 Spatio-Temporal Memory

结果：所有难度 success rate 都变成 0。

原因：任务需要记住菜谱、历史行动和环境状态。没有记忆，agent 连简单任务都无法完成。

### 去掉 Summarizer

简单任务影响较小，复杂任务掉得明显。

原因：长历史会稀释有用信息，让 LLM 更容易被无关内容干扰。

### 去掉 Spatial Memory

复杂任务明显变差。

作者认为 spatial memory 给 agent 提供了类似 “sense of space” 的能力。

值得注意：论文也指出，错误或不准确的 spatial belief 可能误导模型，有时甚至比没有 spatial belief 更糟。

这对我们非常重要：我们的 evidence graph 必须重视准确性，不能乱抽取。

### 去掉 Relation Aggregator

性能中等程度下降。

说明 LLM 能读 triples，但把 triples 转成自然语言 belief 更有帮助。

对我们而言，证据图也不应该只输出 JSON triples，最好转换成：

- 时间线摘要；
- 嫌疑人证据卡片；
- 地点/物品状态表；
- 矛盾清单。

### 去掉 Critic

复杂任务表现显著下降。

作者观察到：LLM 做 critic 往往比做 planner 更强，因为 critic 更像分类任务，判断一个行动是否正确比生成完整计划容易。

对我们而言，这很关键：

> 让 LLM 直接找凶手可能难，但让它检查“这个解释是否违反证据图”可能更可靠。

## 和我们课题的关系

STMA 和我们的研究方向高度相关，但任务不同：

| STMA | 我们的课题 |
|---|---|
| Embodied task planning | Detective narrative reasoning |
| TextWorld cooking task | 侦探故事 / 案件推理 |
| Temporal memory | 时间线 / 证词顺序 / 事件历史 |
| Spatial memory KG | 人物-地点-物品-事件证据图 |
| Planner | 推理器：生成嫌疑人解释 |
| Critic | 验证器：检查时间线、证据、矛盾 |
| Success Rate / Average Score | 答案准确率 / 证据覆盖 / 矛盾识别 / token cost |

我们可以把 STMA 改造成文本侦探推理版本：

1. **Temporal Evidence Memory**：记录故事事件顺序。
2. **Spatial/Object Evidence Graph**：记录人物、地点、物品状态。
3. **Evidence Aggregator**：把图转成模型容易读的证据摘要。
4. **Reasoner-Critic**：先生成推理，再用证据图检查。

## 对开题报告的可用说法

可以这样写：

> STMA 表明，在长程动态任务中，显式的时空记忆和 planner-critic 闭环可以显著提升 LLM agent 的任务完成能力。虽然其应用场景是 embodied task planning，但其 temporal belief、dynamic KG spatial memory 和 critic-based verification 思想，可以迁移到侦探叙事推理中，用于构建时空证据图并验证模型推理是否与证据一致。

## 对我们研究问题的影响

读完 STMA 后，我们的研究问题可以进一步具体化：

> 在侦探叙事推理中，显式的时空证据记忆和 critic-style verification 是否能比普通 CoT 更有效地提升 LLM 的答案准确率、证据一致性和推理成本效率？

可以拆成：

1. Evidence Graph 能否提升最终答案准确率？
2. Evidence Graph 是否能减少时间线、地点、物品状态错误？
3. Critic-style verification 是否能发现并修正错误推理？
4. 结构化记忆是否比简单增加 CoT tokens 更高效？

## 审稿人视角

这篇论文会让我们的选题更有支撑，但也带来一个挑战：

**别人已经证明时空记忆对 embodied planning 有用。我们的创新不能只是“也做一个时空记忆”。**

我们的差异应放在：

1. 从 embodied navigation 迁移到 detective narrative reasoning。
2. 从 spatial memory 扩展到 spatio-temporal evidence graph。
3. 重点评估 evidence consistency，而不只是任务完成率。
4. 结合 test-time improvement，研究不同 thinking budget 下的推理效率。
